05# Adding Package with IMAGE_INSTALL

In this tutorial, you will learn how to add packages to your Yocto-based image. You will discover how to check if a package is available in your layers, add it to your Yocto build, build the image with the added package, and verify if the package is installed on the target device.


Now before adding a package to your Yocto image, you need to check if the package is available in your layers. This tutorial will guide you through finding, adding, and verifying packages in your Yocto build.

How to Find Packages

To find a package, you first need to set up your build environment. Follow these steps:

Navigate to your poky folder. Now source the build environment using the following command: 

source oe-init-build-env

Once the environment is set up, you can use the following commands to check if a package is available: 

bitbake-layers show-recipes
bitbake-layers show-recipes python3
bitbake-layers show-recipes git

How to Add a Package

If the package is available, you can add it to your image by editing the local.conf file. Add the following line to include the package: 

IMAGE_INSTALL:append = " package_name"

For example: 

IMAGE_INSTALL:append = " python3"
IMAGE_INSTALL:append = " git"

Building the Image

After adding the package to local.conf, you need to rebuild the image. Use the following command: 

bitbake core-image-full-cmdline

Verifying the Packages

Once the image is built, flash it to your BeagleBone Black using the steps from Tutorial 01. After booting the BeagleBone Black, verify that the packages are installed by running the following commands: 

# General syntax
# which <package_name>

which python
# Output: /usr/bin/python

which git
# Output: /usr/bin/git

If no output path is displayed, it means the package has not been installed.
